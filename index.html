Introduction

ModalCards.js is a OOP lightweight, configurable modal library written in pure JavaScript.

Main features of the library:

  • Interactions handled by the library:
    • Closing modal on overlay click
    • Closing modal by pressing on close icon
    • Minimize modal to try
    • Drag modal
    • Resize modal
    • Add buttons and Icons
  • Follow Object Oriented Programing conventions.
  • Implement plugin architecture principles offering the user the opportunity to extend the program without touching the core modal library.
  • Allow custom dom event management.

Usage

Designed to be easy to use, it does most of the heavy lifting behind the scenes and exposes a simple api to interact with the dom.

1. Create a customizable model object.

2. Use the init() method to initiate the desired HTML structure.

2. Insert your content to the modal with API methods.

3. Add plugins as needed.

Configuration

The modal object is configurable upon object instantiation and in runtime via API. This allows you to set custom callbacks and control behaviour of the modal.

1.Customization is possible through the 'options' property of the Model object. Example of the JSON object definition:
let simpleModal = new Modal(
options = {
onClose: function(f, b) { // [1]
console.log('modal close');
},
onOpen: function() { // [2]
console.log('modal open');
},
beforeOpen: null, // [3]
beforeClose: null, // [4]
header: true, // [5]
footer: false, // [6]
cssClass: ['custom-class-1', 'custom-class-2'], // [7]
closeLabel: 'Close', // [8]
closeMethods: ['overlay'] // [9]
}
);
[1] onClose function
This is fired when the modal is closing. The anonymous function is completely customizable and can accept any parameter and return any result.
[2] onOpen function
This is fired when the modal is opening. The anonymous function is completely customizable and can accept any parameter and return any result.
[3] beforeOpen function
This is fired before the modal is opening. The anonymous function is completely customizable and can accept any parameter and return any result.
[4] beforeClose function
This is fired before the modal is opening. The anonymous function is completely customizable and can accept any parameter and return any result.
[5] header boolean
This creates the Html element where header of the modal will be placed as a child of the modal container. The default value is false
[6] footer boolean
This creates the Html element where footer of the modal will be placed as a child of the modal container. The default value is false
[7] cssClass Array
Custom classes to be applied when modal is open. The class is attached to the overlay property of the modal object. Default class is '.overlay-class'
[8] closeLabel string
When 'button' is chosen as an option for 'closeMethods' key, this will be it's label.
[9] closeMethods Array
Options of closing the modal by the user. The default value is '['overlay', 'button']'
2. Create the modal HTML structure:
simpleModal.init();
3. Add content with API methods
simpleModal.setHeader('Simple Modal');
simpleModal.setContent('My modal content from API method!
Tap outside the dialog to close the Modal.');

You can also manipulate the modal by using the following API methods:

Name Param Type Description
setHeader(header) String Set modal header
setContent(content) String Set modal content
setFooter(footer) String Set modal footer
openModal() Open modal
closeModal() Close modal
minimize() Minimize the modal window to try
maximize() Maximize the modal window to default size and position
addHeaderBtn(label, cssClass, callback) String, String, function() add Header Buttons. Add costum label, css class and callback function - from API functions or create your own costum callback
addFooterBtn(label, cssClass, callback) String, String, function() add Footer Buttons. Add costum label, css class and callback function - from API functions or create your own costum callback
extend(plugin) String Load plugins to modal object. Use JS import() function

PlugIns

4.You can add our plugin files or create your own. This allow for light core file. To include plugin files:
1. Include the plugin file in the project folder.
2. Use javascript import() method to load the plugin file in runtime and apply it to relevant instance of the Modal class.
3. API Modal methods are accessible to instance from the import() function.

Drag and Drop Plugin

// add drag plugin
import('./modalDragPlugin.js').then((response) => { dragModal.extend(response.ModalDragPlugin);
dragModal.init();
});

Resize Plugin

// add resize plugin
import('./modalResizePlugin.js').then((response) => { resizeModal.extend(response.modalResizePlugin);
resizeModal.init();

// use Modal API
resizeModal.setContent('my resized modal!');
});
